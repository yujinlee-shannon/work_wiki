# 바이브캐치 결정 사항

## 알림 scope와 수신 범위

- 개인 전역 알림과 워크스페이스 단위 알림을 구분합니다.
- 개인 전역 알림 예시는 사용자 초대, 워크스페이스 초대 수락 요청, 본인 계정 관련 알림, SNS 연동 상태 변경 등입니다.
- 워크스페이스 단위 알림 예시는 리포트 생성 완료, 이슈 알림, 언급량 알림, 워크스페이스 참여 완료, 권한 변경, 크레딧 사용량, 공지/점검/정책 변경 등입니다.
- 워크스페이스 단위 기능에서 발생한 알림은 해당 워크스페이스 소속 멤버 전체에 동일하게 노출하는 방향입니다.
- SNS 연동 상태 변경처럼 등록자만 처리할 수 있는 이슈는 등록자와 워크스페이스 관리자에게 전달하는 방향입니다.

## 리포트 기능 일정

- 키워드별 일간/주간/월간 요약 리포트 기능은 1차 오픈 이후 추가합니다.
- 하이닉스 측에는 구체 일자보다 8월 중 바이브캐치에서 사용할 수 있을 것으로 안내되었습니다.
- 개발팀은 2026-08-01 오픈 이후 가능한 일정으로 기능을 추가하는 방향입니다.

## 기기 한도 초과 후 재로그인 흐름

- 기기 한도 초과 후 기존 비밀번호를 FE 메모리에 보관해 재전송하는 방식은 지양합니다.
- 신규 `POST /auth/login/continue` 엔드포인트를 추가하는 방향입니다.
- `pendingToken`은 httpOnly 쿠키로 전달합니다.
- continue 요청 body에는 `terminateSid`만 포함하고, 이메일/비밀번호는 다시 보내지 않습니다.
- pendingToken 쿠키는 HttpOnly/Secure, 제한된 Path, 짧은 Max-Age로 운영하는 방향입니다.
- 성공 시 Redis 토큰 삭제와 쿠키 즉시 만료가 필요합니다.

## 기기 관리 화면 상태와 날짜 표기

- 기기 관리 화면은 로그인 시 기기가 최소 1개 등록되므로 빈 상태는 별도 설계가 필요하지 않습니다.
- 로딩/에러 상태도 메뉴 단위 별도 시안은 필요하지 않은 것으로 정리되었습니다.
- 로그인 날짜/시간 표기는 제품 전반 기준에 맞춰 `yyyy-mm-dd hh:mm` 형태로 맞춥니다.

## AI 인사이트 예상 언급량 기준

- 현재 언급량은 이번 주 현재까지의 누적 언급량입니다.
- AI 예상 언급량은 다음 주 예상 언급량입니다.
- 예측 근거 데이터는 진행 중인 이번 주를 제외하고, 직전 완료된 4개 주차 데이터를 기준으로 합니다.
- 조회 기간과 AI 인사이트 예측 영역은 목적이 달라 독립적으로 동작하는 방향입니다.

## 동기간 대비 증감률 표현

- 사용자가 특정 기간을 조회한 경우, 선택한 기간 일수만큼 직전 동일 길이 기간과 비교합니다.
- 빠른 날짜 선택이 최근 7일, 최근 14일, 최근 30일처럼 명확한 경우에는 전주 대비, 전반부 대비, 월초 대비 등 익숙한 표현을 사용할 수 있습니다.
