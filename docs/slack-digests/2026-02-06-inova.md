# 2026-02~06 아이노바 Slack 정리

## 요약

`#aiagent_기획-개발` 채널에서는 아이노바의 기능 개발, QA, 모델 운영, 공지사항, 계정 운영이 함께 논의되었습니다. 주요 업무 흐름은 폴더 공유/아티팩트 공유 기능 개발, 공지사항 정책 정리, 신규 LLM 모델 관리, 휴직자 계정 운영, 검증/상용 반영입니다.

## 주요 결정

- OpenAI Key 차단 이슈 이후 OpenAI 모델을 우선 비활성화하고, title 요약 모델도 Gemini 계열로 교체하는 방향입니다.
- 신규 모델은 사고 과정 노출 여부와 추론 모델 여부에 따라 카테고리를 조정합니다.
- 휴직자 계정은 별도 개발 없이 관리자 페이지 토글 OFF로 수동 운영합니다.
- 공지사항 Dot은 등록일 14일 이내이면서 미열람인 글에만 표시합니다.
- 폴더 공유는 개인/팀 공유를 고려하고, 공유받은 폴더는 읽기 전용 대화로 제한합니다.
- 아티팩트 노출 속도 개선은 LLM 1회 호출 구조로 변경하고, 비용을 고려한 모델 선택을 병행합니다.

## 주요 이슈

- OpenAI Key 차단 원인 확인과 방어 로직 적용이 필요합니다.
- LLM 사용량 분석에서 Input 비중이 높아 비용 모니터링이 필요합니다.
- 서비스 지연/작동 불가 상황은 재기동으로 복구되었으나 원인 확인이 남았습니다.
- 공지사항 읽음/Dot 로직은 LNB와 리스트 기준을 일치시켜야 합니다.
- 폴더 공유 상용 반영 후 모델 변경 시 대화 위치와 삭제 권한 개선이 추가로 반영되었습니다.
