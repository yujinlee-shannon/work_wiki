## 기록 방식

Slack은 원문과 증거로 두고, 위키에는 업무 판단에 필요한 결론만 남깁니다.

- 결정 사항: 나중에 “왜 이렇게 했는지” 확인할 수 있게 기록합니다.
- 이슈: 장애, 버그, 확인 요청, 정책 질문을 분리해 추적합니다.
- 타임라인: 화면설계서 버전, 배포/반영, 주요 논의 일자를 남깁니다.
- 운영 메모: 반복 안내가 필요한 정책과 절차를 모읍니다.
