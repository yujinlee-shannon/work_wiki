# 위키 운영 규칙

## Slack에서 위키로 옮기는 기준

위키에 남길 항목:

- 결정된 정책
- 일정과 범위가 정해진 기능
- 반복해서 묻는 질문
- 장애/버그 원인과 처리 결과
- 외부 감사, 보안, 운영 기준
- API/화면설계/데이터 계약처럼 구현에 영향을 주는 합의

위키에 남기지 않을 항목:

- 잡담
- 단순 확인 반응
- 임시 계정, 비밀번호, 내부망 주소
- 개인정보 또는 민감정보
- 결론 없는 중간 논의 전체

## 문서 작성 형식

결정 사항은 아래 형식을 우선 사용합니다.

```md
## 결정 제목

- 결정 내용
- 배경
- 영향 범위

원문: Slack 링크
```

이슈는 아래 형식을 우선 사용합니다.

```md
| 일자 | 주제 | 내용 | 상태 |
| --- | --- | --- | --- |
```

## 갱신 주기

- 매주 1회 Slack 주요 스레드를 확인해 `slack-digests/`에 주간 정리를 추가합니다.
- 결정된 내용은 `projects/*/decisions.md`로 승격합니다.
- 해결된 이슈는 `projects/*/issues.md`의 상태를 갱신합니다.
